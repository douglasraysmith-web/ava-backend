# AVA Visible Being + Fast Runtime

Ava needs to feel present, not like a blank chatbot. The visible being layer reports current state: idle, listening, thinking, answering, continuing, gated, and owner-only. The fast supervisor runtime answers common AV definitions, troubleshooting starts, planning paths, driver/source lookup requirements, and customer-data boundaries without waiting for full provider dispatch.

This improves speed because the first useful answer can appear immediately, while the deeper model/source/database work runs only when needed.
